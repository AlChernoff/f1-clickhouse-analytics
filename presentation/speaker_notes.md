# Текст защиты: F1 ClickHouse Analytics

## Слайд 1. Титульный
Здравствуйте. Меня зовут Александр Чернов. Мой проект называется «Аналитика данных Формулы 1 с помощью ClickHouse». В проекте я построил полный аналитический pipeline: от CSV-данных до технического мониторинга и BI-дашборда.

## Слайд 2. Цель и задачи
Цель проекта — сделать воспроизводимую аналитическую систему для данных Формулы 1. Для этого нужно было решить четыре задачи: загрузить данные, организовать хранилище в ClickHouse, построить аналитические слои через dbt и визуализировать результат в Grafana и Superset. Главный критерий готовности — проект можно поднять с нуля командой `make demo`.

## Слайд 3. Технологии
В качестве хранилища я выбрал ClickHouse, потому что он хорошо подходит для аналитических агрегаций. Python используется для replay-loader, который имитирует потоковую загрузку исторических CSV. dbt отвечает за staging, DWH, marts и тесты качества данных. Grafana показывает технический мониторинг pipeline, а Superset — бизнесовую аналитику. Вся инфраструктура описана в Docker Compose.

## Слайд 4. Архитектура
Архитектура начинается с CSV dataset. Python loader читает файлы и загружает данные батчами в ClickHouse. В ClickHouse есть отдельные базы raw, dwh, marts и monitoring. dbt строит аналитические модели, а dashboards подключаются уже к готовым таблицам. Это позволяет показать полный путь данных от источника до визуализации.

## Слайд 5. Слои данных и дедубликация
Raw-слой хранит данные близко к источнику. В staging выполняется очистка и логическая дедубликация. DWH содержит измерения и факты, например dim_drivers и fact_race_results. Marts — это готовые витрины под BI, например performance пилотов и команд. Для дедубликации используются два уровня: ReplacingMergeTree в raw и row_number в dbt staging. BI работает только с marts.

## Слайд 6. Мониторинг и BI
В проекте есть два типа визуализации. Grafana показывает технический мониторинг загрузки: количество загруженных строк, успешные и неуспешные batch-и, длительность загрузки и последние события pipeline. Superset показывает бизнесовую аналитику: топ пилотов, топ команд, динамику сезонов, быстрые пит-стопы и времена кругов.

## Слайд 7. Что получилось
Основной результат — воспроизводимый demo-flow. Команда `make demo` пересоздает окружение, загружает данные, запускает dbt, импортирует Superset dashboard и выполняет проверки. Команда `make demo-show` красиво показывает текущее состояние проекта в терминале. После этого можно открыть Grafana и Superset и показать dashboards.

## Слайд 8. Выводы и развитие
Цель проекта достигнута: данные загружаются, трансформируются, мониторятся и визуализируются. Самыми сложными частями были типы данных в CSV, настройка прав ClickHouse user, импорт Superset dashboard и стратегия дедубликации. В дальнейшем проект можно развивать: добавить Prometheus exporter, Kafka для настоящего streaming, CI для dbt tests и cloud deployment.
